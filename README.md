London Transit SQL Analysis (TfL RODS Dataset)

📊 Overview
This project analyzes ridership patterns in the London Underground using Transport for London (TfL) Rolling Origin and Destination Survey (RODS) data.

🎯 Objective
To uncover trends in passenger behavior, including:
- Peak travel times
- Most common travel purposes
- Zone-based ridership patterns

🛠️ Tools Used
- SQL
- Dataset: TfL RODS (London Underground)

🔍 Key Questions
- When are the busiest travel periods?
- Why do passengers use the Underground?
- How does ridership differ by zone?
- How do travel patterns vary for tourism vs commuting?

📈 Key Insights
- Total daily journeys exceed 4.8 million, showing heavy reliance on the system
- PM Peak (4–7 PM) has the highest passenger volume
- Over 50% of journeys originate in Zone 1, highlighting central London as a major hub
- Work-related travel dominates, confirming commuting as the primary use case
- Tourism-related travel is more evenly distributed throughout the day
- Outer zones (2–5) show more trips originating from home, while Zone 1 shows more work-related origins

📂 Project Structure
- `queries.sql` → SQL queries used for analysis
- `insights.md` → detailed analysis and interpretations

🚀 Skills Demonstrated
- SQL Aggregations (SUM, GROUP BY)
- Data segmentation and filtering (WHERE)
- Multi-column grouping
- Analytical thinking and pattern recognition

💡 Business Recommendation
Increase service frequency during peak hours—especially in Zone 1—and optimize scheduling to better handle commuter traffic while maintaining flexibility for tourism-related travel.
